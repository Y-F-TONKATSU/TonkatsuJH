いかなる夢に基づいて
Web サイトを作るか


一つの夢としての
セマンティック・ウェブは、
(XHTML が普及しなかったりとか)
短期的には行きつ戻りつしていても、
時間スケールを大きくとって見れば
確実に実現に
向かっているように思われる。
新しい Web 標準の技術と
モダンブラウザは
徐々に Web の
インタラクションの世界に
秩序を取り戻しつつある。
急速に普及した、
モバイル Web の世界でも、
標準的な UI が
日々確立してきている。
私もそれは素晴らしく
良いことだと思っていて
反抗したいわけではないが、
かといって格別
現状の進化の方向に
貢献したいわけでもなく、
現実的な要求に沿う範囲内で、
別の方向への進化の可能性を
探るような気持ちで
Web サイトを作っている。
色々な種類の夢が
共存できるべきだとだけは
わりとはっきりと考えている。
Flash 製の前作、
キネマぽんぽこ (2008～)では
「何か触ると動く」面白さを
追求してみるつもりだった。
(「やっくんドゥードゥルドゥーの
省略形としての躍動感」
という標語は
このコンセプトを表している。)
モバイル機器の
バッテリーという
新たな十字架を
背負い込むことで、
今やこの手の夢は
17 歩ほど後退してしまった。
プログレッシブ・エンハンスメントを
徹底すれば、
こういうのも
まだ実現可能なのだろうが、
多くの人が
iPhone で Web を
見ているという事実と、
Web 標準の技術及び
検索エンジンの進化を受けて、
データ構造に関わりのない部分に
装飾的な動きを
付け加えることへの
モチベーションは
(私ですら)
どんどん下がってきている。

棚のインターフェース

"TONKATSU now Making" は
あくまでも
アプリの紹介が主眼で、
たくさんのコンテンツを
載せることを
想定していなかった。
しかし、アプリがそんなに
さくさく作れるもんでも
ないことが分かってくる。
目下やることは勉強で、
それでもなにかやってる感は
出したかったりもして、
SNS に中間制作物を
アップしたりしていた。
それはそれで
直ぐに反応があったりして
いいのだけど、
作ったものを
少しだけまとまった形で
日常的に手軽に
発表することで、
少しづつアーカイブを
積み上げていく仕組みとしての
個人サイトにできないか、
と方向転換した。
動きを抑え、
空間性を抑え、
平面上のシンプルさを捨て、
レスポンシブデザインを導入し、
構造化文書と
検索エンジンと
SNS との親和性を高め、
そして大量のコンテンツを
一面にぶちまける。
インテリア写真集によくある
一面に物が貼ってある壁や
壁一面の広がる棚に
こだわりのあるものを
並べてあるような部屋の
イメージがピッタリだと思い、
『棚』を
コンセプトの中心に据えた。
最初期の棚
色々と棚を試作し、
素材のループなどの
現実的な要求による制約を
くぐり抜けて完成したのが、
現状の棚だ。
棚を作ったの時期に、
わけのわからん個人が
数人で作ったアプリが
良い物になり得るんだろうか、
なんてことも考えだした。
もし良い物になり得るとすれば
それは作った奴が
面白いからそうなる、
ということでしか
有り得ないのではないか、
と今のところ考えている。
もちろん現状が
そうだとはとても言えない。
このサイトも
作った人間と同じで、
悪いところが無数にある。
しかし、
このサイトがダメなのは
作った人間がダメだからだ、
と言えるものができただけでも
とりあえずはよい。
