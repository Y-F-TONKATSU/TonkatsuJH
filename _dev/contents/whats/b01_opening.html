ウィトゲンシュタインとカレー

我々は『頭上の星、内なる道徳律、目前のカレー』に
限りない想いを寄せるアプリ制作集団 "TONKATSU" である。
昔の人はみんなカレーが好きだった。
我々の世代でその偉大な精神を失ってしまうのは
先祖に対して申し訳のないことである。
そのような思いを新たにしたのは
このサイト"とんかつひろば" の
アートコンセプトを決めている最中であった。
アプリの集客とかもろもろのためにサイトを作ろうという話を
オノデン(TONKATSU アプリの美術デザイン担当)としていたとき、
最初はアプリのアイコンと Google Play へのリンクだけが並んだ
シンプルなサイトにしようということになったのだが、
山田.F.とんかつ(筆者) は内心ではそんな案にはもちろん満足しなかった。
それで新宿の紀伊國屋書店などに
面白いアートコンセプトを探しに行った。
そのときにたまたま建築のコーナーで

材質とか空間とか

重要なのは建築（とカレー）というところである。
キネマぽんぽこではアニメ調のベクターグラフィックを使ったが、
今回は物質的なイメージを前に出していこうと決めた。
そもそも、自分が映像の方面でやりたいのが
ハンドメイド感のあるCG みたいなことだった。
Web サイトを作るにあたっても、
CG を使いつつ現実に素朴にモノを組み立てる感覚で
やっていったらいいのではないかと、
建築の本を眺めているうちに思えてきた。

Top Space & Art: Spatial and Graphic Design for Event and Exhibition

とにかく載ってる図版が全部かっこいい。 その後、水戸のイラストレーターの伊敷トゥートさんに会う機会があり、
この際、看板になるような絵を描いてもらうことにした。
描いてもらった絵を見たら、すぐに完成形のイメージが浮かんだ。
自分の作りたいものって、
実際に何かを作ったり、自分の外にあるものを取り入れたりしないと
分かんないものだなーと改めて思った。
もっとも、Web制作というのは、
美術以外にも気を使うことがいっぱいあって、
(実際は大体プログラムのことを考えていた)
いつも結果をみると、アートコンセプトは徹底的に達成できてはいなくて、
今回もご多分に漏れずである。
建築的なものを意識するというのは、
空間的な広がりを感じさせるようなものにする
ということでもあったんだけど
そこには今回は及ばなかった。
擬似3Dで、ダウンロードサイズはできるだけ減らして、
ユーザビリティにも気を使って、
検索エンジンにもある程度フレンドリーに、
と、こう制約が多いとカメラをダイナミックに動かすような演出は
なかなか難しい。
結局、一定の狭い空間の中でものをごちゃごちゃ動かす、
という感じになってしまい、
キネマぽんぽこからあまり進歩していない。
そのへんは、今後アプリの方でやっていきたい。
多様な空間を移動できるゲームとか作ろうと思う。

技術的なこと

技術的なことで新しくやったのは
Flash Player を使わず(オーサリングツールとしての Flash は使った。) 、
JavaScript だけで動かすということ。
ただ、今まで Flash で培ってきたものを全部捨てるのは厳しいので、
CreateJS なるライブラリを使わせてもらった。
CreateJS は Flash のプラグインと JavaScript のライブラリのセットで
Flash ムービーを Canvas と JavaScript に書き出してくれる。
シンボルは JavaScript で書かれた Container クラス(のようなもの)の
サブクラス(のようなもの)に変換される。
Container はそれぞれ Timeline オブジェクトを持っていて
Flash で描いたアニメがクラシックトゥイーンも含めてそのまま再現されて、
gotoAndPlay() とかのおなじみのメソッドも使える。
シンボルのカラー効果とか非クラシックなトゥイーンとか
使えない機能はいくつかあるが、
シェイプトゥイーンやレイヤーマスクは再現してくれてたり、
健闘していると言っていいと思う。
とくに有難いのは Flash の DisplayList の便利さを
ある程度再現してくれていて、
ネストされたインスタンスのマウスイベントも拾ってくれるところ。
素の Canvas でこれをやろうとするとかなり面倒くさい。
(というか、ベクターグラフィックが要らないなら、
Canvas 外で DHTML 的な手法を使ったほうが早い。
この辺が私が Canvas に
「こんなもん Flash の代わりになるかい。」
と思う主な理由。)
ベクターグラフィックについてはほぼそのまま持っていけるし、
プリロードの制御ができる PreloadJS もついてるし、
大変有り難かった。
バグとかブラウザによる動作の違いも多くて大変だったが。
あと、IE で動かないという問題もあって、
これに関してはもう諦めて PHP で弾くことにした。
なんか Web 標準に合わせたせいで、
かえってまともに動く確率下がってる気がする。
DOM とかブラウザごとの実装の違いとか、
CSS の直感的じゃなさとか、
Canvas がやっぱり Flash と比べると全然表現力がなかったりとか、
まだ、ビジュアル主導のサイトを作るなら
相変わらず Flash の方が全然いいと思うけど、
JavaScript は素晴らしく楽しい言語であるとは思った。
ActionScript3 も綺麗に書けていいけど、この自由な楽しさはない。

とんかつのパラドックスという哲学上の有名な難題がある。
何が難しいかというと、それが何なのか筆者もよく知らない。
知っている人を見たこともない。

それじゃあ、なぜそんなことをここに書くのかという問題に至っては、
未だ解決の糸口も見つかっていないのが現状である。
