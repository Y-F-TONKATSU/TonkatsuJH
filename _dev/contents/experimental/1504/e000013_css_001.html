ボックスモデル

HTML の要素は
ブロック要素であっても
インライン要素であっても
ボックスを構成する。
例:
{
	width:10px;
	height:10px;
	padding:10px;
	border:10px;
	margin:10px;
}
のように指定すると、
このようなボックスが
形成される
border も
独自のエリアを
形成する点と、
コンテンツエリアは
余白を一切含まない点は
注意しておきたい。
なお、本記事では、
コンテンツ, padding,
border, margin
それぞれの領域を
コンテンツエリア、
パディングエリア等
「エリア」と呼び、
コンテンツから
マージンまで
全てのエリアを
含めた領域を
「ボックス」と呼ぶ。
また、
あるボックスを
包含する
親のコンテナ要素の
ブロックを、
そのボックスの
包含ブロックと呼ぶ。

ボックスのサイズと
width, margin

上図から分かるように
ボックスのサイズは
コンテンツ, padding,
border, margin
の合計で決まる。
また、
コンテンツエリアのサイズは
width, height プロパティで
決まる。
と言っても、
width, height を
CSS で auto にした場合、
もしくは
この後見ていくように、
width, height を
指定できない
インラインボックスの場合、
width, height の
最終的な値は、
コンテンツのサイズ及び、
margin と包含ブロックのサイズの
関係によって、
動的に変化する。
(そして auto は
初期値なので
指定しなかった場合も
そうなる。)
ここが CSS の
最も理解し難い
ポイントの一つである。
しかし、
width と margin の
関係などは
次回に譲る。
今回はまず、
display:block と
display:inline と
display:inline-block
の違いを見ていく。
ここが理解できれば、
width や margin の
正体も見えてくる。
なお、本記事は
横書きを前提とする。
(とはいえ、
縦書きの場合も
width と height などを
読み替えれば、
そのままである)

ブロックボックスと
インラインボックス

大まかに言うと、
インラインボックスは
1行の中に
いくつも含められる
ボックスであり、
ブロックボックスは
1ボックスで
1行を埋め尽くす
ボックスである。
ではこの場合、
「行」とは何か？
行とは、
包含ブロックの width,
大まかには
親要素の
コンテンツエリアの
横幅のことである。
ボックスを
包含ブロックに挿入しても、
まだ横幅が余る場合、
width か margin を
調整して
余白を埋めてしまうのが
ブロックボックスであり、
余ったスペースに
別のボックスを挿入するのが
インラインボックス
だとも言える。
以下、具体例。

display:block

  • ボックスが包含ブロックの幅を
    100% 埋める。
    他のボックスが横に並ばず、
    前後に改行が入ったような
    見た目になる
  • width, height を指定できる
例:
/* html */
<div class='parent'>
	<p class='a'>ドリンクバーの回し飲みは犯罪なんです！ </p>
	<p class='b'>犯罪なんですよぉォォーー！！！</p>
	<p class='c'>ドリンクバーの回し飲みは！ カッコ悪いんだよ！！</p>
</div>

/* css */
.parent{
	display:block;
	margin:0px;
	padding:0px;
	width:200px;
	height:250px;
	background-color: orange;
}

.a{
	display:block;
	width:auto;  /*最終的な値は 200px (包含ブロックの width と同じ)*/
	height:auto; /*最終的な値は 2行分の高さ*/
	margin:0px;
	background-color: green;
}

.b{
	display:block;
	width:auto; /*最終的な値は 100px (包含ブロックの width から左右 margin を引いた値)*/
	height:auto;
	margin-right:50px;
	margin-left:50px;
	background-color: blue;
}

.c{
	display:block;
	width:100px;
	height:100px;
	margin:auto; /*最終的な左右の margin 値は 50px (包含ブロックの width から 100px を引いて、残りを左右に均等に割り振った値)*/
	background-color: black;
}
ブロックボックスは
width, padding,
border, margin の合計が
包含ブロック
(上の例では
parent クラスのdiv)
の width と一致する。
上の例では、
width を固定値で
指定した場合は
margin が、
margin を固定値で
指定した場合は
width が調整されて、
行(包含ブロックのwidth) を
埋めているのが分かる。
width と margin を
両方固定値で
指定した場合、
margin-right が
勝手に auto になり、
無理やり隙間を埋める。
この width と margin の関係は
重要なところで、
次回にさらに書く。

display:inline

  • 前後に
    他のインラインボックスを
    置くと、
    横幅が足りるなら、
    横に並べられる
  • width, height の指定は無効
  • 複数行に渡ると、
    領域が行ごとに
    分断されたようになる。
例:
/* html */
<div class='parent'>
	<p class='a'>「神は人間があたえられている自由を乱用してドリンクバーの回し飲みをするのを欲してはいない。</p>
	<p class='b'>しかし、神はドリンクバーの回し飲みを妨げないのだ。神は人間が自分で選択してドリンクバーの回し飲みをしないように人間を自由なものにしたのだ。」</p>
</div>

/* css */
.parent{
	display:block;
	width:200px;
	height:200px;
	padding:0px;
	margin:0px;
	background-color: orange;
}

.a{
	display:inline;
	width:auto; /* 無意味な指定 */
	height:auto; /* 無意味な指定 */
	margin:0px;
	background-color: green;
}

.b{
	display:inline;
	width:100px; /* 無意味な指定 */
	height:100px; /* 無意味な指定 */
	margin:0px;
	background-color: blue;
}
width, height の指定は
無効になっている。
横幅は
コンテンツを表示するのに
必要なだけの大きさになり、
包含ブロックの大きさに合わせて、
margin が変わることはない。
包含ブロックの width 分の
横幅を使い果たしたら
改行され、
改行されるごとに
どんどん縦に
伸びていく。
(縦にはみ出した時の
処理は、
包含ブロックの overflow の
値で決まる。)
なお、上の例では
margin も padding も
0 なのに、
ボックスとボックスの間に
小さな隙間が空いているが、
これは本文中の
2つの<p>タグの間の
改行コードとタブが
1つのスペースとして
解釈されているため。 
/* html */
<div class='parent'>
	<p class='a'>ドリンクバーの回し飲みは</p><p class='b'>ゆるさないぞ！</p>
</div>
として、
スペースをなくすと
隙間はなくなる。

display:inline-block

  • width, height を
    指定できる。
  • 包含ブロックを埋めず、
    他のボックスと
    横に並べられる
例:
/* html */
<div class='parent'>
	<p class='a'>インド思想では</p>
	<p class='b'>ドリンクバーの回し飲み</p>
	<p class='a'>は</p>
	<p class='b'>「輪廻(サンサーラ saṃsāra)」と呼ばれており、仏教では苦とされている。</p>
	<p class='c'>仏教の目的は回し飲みの環からの解脱である。</p>
</div>

/* css */
.parent{
	display:block;
	width:250px;
	height:250px;
	margin:0px;
	padding:0px;
	background-color: orange;
}


.a{
	display:inline-block;
	width:auto;
	hegiht:auto;
	margin:0px;
	background-color: green;
}

.b{
	display:inline-block;
	width:100px;
	height:100px;
	margin:0px;
	background-color: blue;
}

.c{
	display:inline;
	margin:0px;
	background-color: black;
}
インラインブロックボックスは
width, height を
指定できるが、
width を固定値で
指定しても、
margin が自動調整されて、
包含ブロックの width を
埋めることはない。
width を
auto にした場合も
コンテンツの
実際の大きさ以上に
コンテンツエリアが
確保されることはない。
逆に言えば、
width が auto で
コンテンツのサイズが
行全体を埋めるほど
大きいなら、
ブロック要素と
変わらない見え方になる。
height を指定したり
改行した結果、
縦に長いボックスに
なった場合、
でかい文字みたいに
扱われる。

置換要素

<img> や<canvas> は
CSS のプロパティではなく、
要素の直接の属性として、
width, height を持つ。
(例:<canvas width='400' height='300' >)
このような、それ自体で
固有のサイズを持つ要素を
置換要素と呼ぶ。
置換要素は基本的に
インラインブロックボックス
のように扱われる。
(要素の直接の属性ではなく)
CSS で、
width, height を指定すると、
ピクセルが
引き伸ばされるような形で
コンテンツ自体が伸縮する。
置換要素の伸縮は
display:inline を
指定していても
発生する。

display:none と
visibility:hidden

ついでに、
display:none
にも触れておく。
display:none にした要素は
ボックスを構成しなくなり、
見た目上は
意味のない要素になる。
visibility:hidden でも、
要素は非表示になるが、
ボックスの構成は
行われるため、
ボックスの大きさ分だけ
隙間があくことになる。
display:none の例:
/* html */
<div class='parent'>
	<p class='a'>ドリンクバーの回し飲みは</p>
	<p class='b'>ゆるさないぞ！</p>
</div>

/* css */
.a{
	display:none;
	background-color: green;
}

.b{
	display:inline;
	background-color: blue;
}
最初の段落は表示されず、
ボックスも構成されない。
visibility:hidden の例:
/* html */
<div class='parent'>
	<p class='a'>ドリンクバーの回し飲みは</p>
	<p class='b'>ゆるさないぞ！</p>
</div>

/* css */
.a{
	visibility:hidden;
	display:inline;
	background-color: green;
}

.b{
	display:inline;
	background-color: blue;
}
最初の段落のテキストの
長さの分だけ
隙間が空いている。
なお、jQuery で
hide() した要素は
display:none になる。
次回は、
width と margin の関係を
詳しく述べる。
display プロパティは
他にも list-item や
run-in などの
値を取ることができるが、
本記事では触れない。
細かいところまで
勉強するなら、
冒頭でも紹介した
この本がおすすめ。
