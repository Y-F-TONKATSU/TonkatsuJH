パーリンノイズをグラフにプロットする

まず単純に、
適当な入力に対する出力を
折れ線グラフにプロットしてみます。

「入力値を与えると
0-1 の間の値を返す関数」を
引数として与えると、
Canvas にグラフを
描いてくれるクラスを
作ってみました。

描画メソッドの要点だけを
以下に示します。
(コード全体は git-hub で公開しています。) 

drawGraph:function(out, progress){
	
	this.init();
	
	var s = 0;
	var progress = progress || this.DEFAULT_PROGRESS;
	
	var pv;
	
	this.play(_.bind(function(){
		
		this._graphics.setStrokeStyle(this._option.lineWidth);
		this._graphics.beginStroke(this._option.color);
		
		var v = out(s) * this._width;
		
		if(_.isUndefined(pv)){pv = v;}
		
		var progressRate = Math.pow(progress, -1);
		
		this._graphics.moveTo(s * progressRate, pv)
		.lineTo((s + progress) * progressRate, v);
		
		pv = v;
		
		s += progress;
						
		this._stage.update();
		
		if(s * progressRate >= this._height){this.stop();}
		
	}, this));
	
},

これにいくつかの関数を与えて、
違いを見てみます。

まっすぐな線

まずは
まっすぐな線を
引かせます。
入力値として
「常に 0.5 を返す関数」
を与えます。

↓こんな感じ。 

var graphUtil = new GraphUtil('#contMainCanvas');

graphUtil.drawGraph(function(input){
	return 0.5;
});

線が引けました!
…面白くも
なんともないですね。
(でもここまで来るのも
けっこう大変だった。
Processing の話の早さの
有り難みがよく分かる。)

ランダムな折れ線

次に入力値として
JavaScript の
ビルトイン関数の Math.random を
与えてみます。 

graphUtil.drawGraph(Math.random);

ギザギザの激しい線になります。

パーリンノイズ

ではいよいよ、
パーリンノイズを
使ってみます。 

graphUtil.drawGraph(noise);

有機的な線が引けます。
これを別の描画系の関数と
混ぜてみます。

円を描く

GraphUtil に
円を描くメソッドを追加してみます。

「入力値を与えると
0-1 の間の値を返す関数」を
引数として与えると、
出力値に応じて、
円が歪みます。

これにいくつかの関数を与えて、
違いを見てみます。

きれいな円

まずは
常に0.5 を返す
関数から。 

graphUtil.drawCircle(function(inpput){
	return 0.5;
});

機械かテヅカ先生か、
って感じの円が
描けます。

ランダムに歪む円

次に Math.random を
試します。 

graphUtil.drawCircle(Math.random);

無茶苦茶ですが、
これはこれで面白い。

歪んだ円

そしてパーリンノイズ。 

graphUtil.drawCircle(noise);

いい感じに
下手くそになりました。
放置して何周かさせると、
線と線が重なって
少し美しいです。

こういう有機的な変化を
作り出すのに、
パーリンノイズは有効。
…ということで、
NoC でも使われるようです。

ノイズの次元

ノイズには次元の概念があります。
NoC でもジェネラティブアートでも
グレースケールでの表現の例が載っていて、
分かりやすいので、
再現してみます。

1D ノイズのグレースケール表現

一次元の場合は
こうなります。
微妙な陰影がついています。
折れ線グラフの例が
上の方にありますが、
あちらでの高さが
こちらでは線の明るさで
表現されているわけですね。

2D ノイズのグレースケール表現

二次元の場合。
横だけでなく、
縦方向にも
微妙な変化が現れます。
