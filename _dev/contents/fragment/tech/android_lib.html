layout_weight の設定

レイアウトが子の要素を配置するルールについてのメモ。 まずは、 width, height の値を優先する。 その上で余った幅があれば、 子の要素の layout_weight の値に応じて 余ったスペースを振り分ける。 layout_weight にはどんな浮動小数点の値を振ってもいい。 CSS の％指定みたいに、 割合だけでレイアウトを決めたい時は 全ての子の要素の width, height を 0dp に設定すると、 単純に layout_weight の比率でレイアウト幅が決まる。

Task と Process

デバイスの Recent ボタンを押すと
表示されるリストに
並んでいる単位が Task と考えてよい。
例えば、あるアプリAが
暗黙的インテントで
カメラアプリBを起動した時、
B も A が含まれるタスクに含まれる。
しかし、B の Process は
独立した B の Process である。
(フラグ Intent.FLAG_ACTIVITY_NEW_TASK を
インテントに含めることで、
独立したスクにすることも出来る。)

R.java が自動生成されない時の対処

R.java が生成されていても、
Android.R と
読み違えられていることもある。
とりあえず、エラーログをよく見る。
以下、過去に上手く言った対処の例。

1. Eclipse のメニューから
Project > Clean.

2. Android SDK Manager を起動して、
SDK を最新に更新してみる。

更新は一回では完了しないこともある。
Android SDK build-tools が
全て最新になるまで更新する。

3. サポートライブラリ関係で
エラーが出ているせいで
R.java が生成されなくなるがある。
Build Path を見直す。

4. ターゲット API 関係で
エラーが出ている可能性もある。
プロジェクトのプロパティの
Android の項から、
ビルドターゲットを変えてみる。
