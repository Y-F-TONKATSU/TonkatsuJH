Android SDK と Eclipse

最近はSDK がバンドルされた Eclipse のパック
なんてのが公開されてて、開発環境のセットアップは簡単。
DL して解凍して適当なディレクトリに置くだけ。

ただし SDK Manager でのパッケージのインストールには
少し罠がある。
インストールしたいパッケージにチェックを入れて、
インストールボタンを押すだけなので、便利なのだけど、
一度インストールボタンを押しただけでは
うまくインストールされてないことがわりとある。

特に Android 4.2.2(API 17) をインストールするときは
一度 Android 4.2.2(API 17) をインストールした後に
Android SDK Build-Tools もインストールしないと
gen ディレクトリ内の R クラスがビルドされなくなるので要注意。
なんというか、ちょっとしつこめにインストールするのがコツである。

Eclipse の独特の概念とか

ワークベンチ

画面全体のこと。
名前が付いている意味がよく分からない。

パースペクティブ

ウィンドウのレイアウトのセットのこと。
Adobe ツールでいうところのワークスペースと同じ。

ワークスペース

作業用フォルダ。

これがどういう扱いなのか最初よくわからなかった。
アプリごとにワークスペース変えたほうがいいの？ とか、
ワークスペース内に開発するプロジェクト全部入れたほうがいいのか？
とか、色々試してしまった。

ポイントはワークスペース毎に設定を保存できる、
というところかと思う。
ワークスペースは基本固定でよい。
プロジェクトによっては違った設定を使いたいことがあるかもしれないので、
そのときにはワークスペースを変えればよい。

プロジェクトの置き場所はワークスペースの中でなくてもよさそう。
ワークスペースの外にある既存のプロジェクトを開くには、
File -> New -> Other -> Android -> Android Project from Existing Code
で OK.

Eclipse のショートカット

Ctrl + Shift + O で import 文の整理、は感動的に便利だ。
たまに違ったものを挿入してくれたりするが。
特に自動生成される R クラスの代わりに Android.R がインポートされて
コードが真っ赤になったアンドロイダーは数多いようである。
ショートカットでは ctrl + D, alt + 矢印,
ctrl + alt + 矢印あたりもよく使う。

完全に静的な型指定

Java では変数は必ず型指定する。
面倒だしやたらとコードが長くなるが、
コンパイルエラーのチェックやコード補完が確実に動作したり、
色んなことを Eclipse が勝手にやってくれるので便利。

変数の初期化

クラスのメンバ変数は作成時に自動的に初期化される。
しかし、ローカル変数は必ず初期化しないといけない。

コンストラクタに関すること

コンストラクタは必ず返り値なしである。
新しいオブジェクトは基本 new を使って作成する。
JavaScript のコンストラクタのようなややこしいことはない。

コンストラクタの宣言を省略すると
デフォルトコンストラクタが自動で作成される。
デフォルトコンストラクタは引数をとらず、
コードブロックにはスーパークラスのコンストラクタの呼び出しのみを含む。
